Introduction

The DCS API is a C++ software library developed by the Tactical Electronic Warfare Division at the Naval Research Laboratory. The API is the software implementation of a system for sending data to Simdis for display, distributed as part of the Simdis 3D Visualization and Display package.  The API transmits predefined data types to Simdis using two standard Internet Protocols (IP).  These Internet Protocols are the Transmission Control Protocol (TCP) and the User Datagram Protocol (UDP).

TCP is a protocol for reliably transmitting data from one point to another.  All data sent via TCP is guaranteed to arrive at its destination, in the order in which it was sent.  This protocol requires a significant amount of overhead to reliably deliver information in sequence, and no one piece of information can be delivered before any of its predecessors.  Examples of applications that utilize the Transmission Control Protocol are web browsers and email programs.  DCS uses TCP to transmit important information such as initialization and state modification data types.  

UDP is a non-reliable Internet Protocol that uses what can be described as a "fire and forget" approach to data transmittal.  UDP data is not guaranteed to arrive at its destination in sequence, or even to arrive at its destination at all.  Some examples of applications that utilize the User Datagram Protocol are network timeservers, domain name servers, and network video games.  DCS uses UDP to transmit information such as position updates, which are only valid during a specific time frame.  This type of data typically arrives at a very high rate, and the infrequent loss of data is negligible when compared with the significant amount of overhead required to reliably transmit the data in sequence.

Features

Current Features

  •  Object-oriented (implemented in C++)
  •  Cross platform - currently runs on:
    •  Linux - g++ 3.x, 4.x
    •  Windows - VC++ 8.0 (2005), 9.0 (2008), 10.0 (2010)
  •  Data Display Objects:
    •  Platforms.
         Platform display objects represent entities such as aircraft, ships, and buildings
    •  Beams.
         Beam display objects can be used to visualize data from objects such as antennas, FOVs, and RF/IR seekers.
    •  Gates.
         Gate display objects can be used to visualize data from objects such as range track gates and search volumes.
    •  Generic Data.
         Generic display objects can be used to textually display any user-generated data.
Software Requirements and Recommendations

DCS is supplied as source code only. This means that you have to compile the package for yourself.  The following is a list of other software you will need or is recommended to build this package:

This library is only distributed in C++ form. To use this library, you must build it on your own machine. The exact steps to build the source may vary based on the operating system and compiler, but the overall method is the same. This page describes the steps required for the Plug-in API version 1.11 and later.

  1. Download and extract the DCS2 Library ZIP or TGZ from the SIMDIS web site.
  2. Run CMake to configure and generate either a solution file (Visual Studio) or makefile (UNIX makefiles)
    • Start CMake
      • Windows users should find a shortcut in their start menu after installation
      • Linux users can run cmake-gui for the X-Windows interface, or ccmake for the curses interface
    • Locate the DCSSoftware/ directory in the field "Where is the source code"
    • Specify a build location in "Where to build the binaries". Note that CMake supports and encourages out-of-source builds, but it is not required.
    • Click Configure. You may be asked to select your compiler version
      • Windows users may wish to select one of:
        • Visual Studio 9 2008 (win32_vc-9.0)
        • Visual Studio 9 2008 Win64 (win64_vc-9.0)
        • Visual Studio 10 (win32_vc-10.0)
        • Visual Studio 10 Win64 (win64_vc-10.0)
      • UNIX users may wish to select Unix Makefiles
    • Optionally set the CMAKE_INSTALL_PREFIX variable. If run, the "make install" will copy headers and libraries into this location. You can ignore this if you do not plan on compiling the install target.
    • Click Configure again to lock in any changes, then click Generate.
  3. On Windows, a solution file is generated in the directory specified in "Where to build the binaries." Run Visual Studio to build.
  4. On Linux, makefiles are generated in the directory specified in "Where to build the binaries." Run make to build.

Programs designed to work with the DCS API are required to include the 'DCS.h' header file.  This file includes all of the necessary DCS, NET, and other miscellaneous header files required for the implementation of a DCS application.  These header files define all of the types and functions provided by the DCS API.   Any C++ source file that makes use of the DCS API must include this line:

#include "DCS.h"

Programs designed to work with the DCS API must also be linked with the proper DCS and NET library files.  For Microsoft Windows based applications, these libraries are DCS.lib and NET.lib, and for UNIX based applications these libraries are libDCS.a and libNET.a.  Microsoft Visual Studio users can link their applications with these libraries by adding them to the "Object/library modules" text field from the "Link" tab of the project settings dialog.  UNIX makefile users can link their applications with these libraries by adding -lDCS and -lNET to the linker options.  As an example, we have provided two sample server programs and a sample client program.  These programs are found under DCS2 in the following sub-directories:

  •  examples/sclient
  •  examples/scserver
  •  examples/sserver
